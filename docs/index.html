Abstract

The Problem: AI systems require structured context to operate effectively, but current tools for creating and maintaining that context lack semantic structure. Engineers manually track entities, relationships, and gaps in knowledge using plain text files, wikis, or note-taking apps that don't understand the semantic meaning of content.

The Solution: Pedantic Raven is a terminal-based semantic memory editor that analyzes text in real-time to extract entities, relationships, typed holes (knowledge gaps), and dependencies. Built with Go and the Bubble Tea framework, it provides a responsive TUI that integrates deeply with the mnemosyne semantic memory system for persistent storage and retrieval.

Key Innovation: Real-time semantic analysis as you type, with automatic fallback between ML-based extraction (GLiNER, 85-95% accuracy) and pattern matching (60-70% accuracy, always available). This hybrid approach ensures the system remains functional even when external services are unavailable‚Äîa design principle of graceful degradation.

System Architecture

Pedantic Raven is built with Go and the Bubble Tea framework, implementing Elm Architecture for predictable state management. It integrates with mnemosyne via gRPC for persistent storage and optionally uses GLiNER for ML-based entity extraction.

Getting Started

Quick Start

# Clone repository
git clone https://github.com/rand/pedantic_raven.git
cd pedantic_raven

# Build
make build

# Run (local-only, no mnemosyne)
./pedantic_raven

# Run with mnemosyne integration
MNEMOSYNE_ENABLED=true MNEMOSYNE_ADDRESS=localhost:50051 ./pedantic_raven

Configuration

Configure via config.toml or environment variables:

[gliner]
enabled = false              # Set true if using GLiNER service
service_url = "http://localhost:8765"
timeout = 5
fallback_to_pattern = true

[mnemosyne]
enabled = false              # Set true if using mnemosyne server
address = "localhost:50051"
timeout = 10
max_retries = 3

Positioning

Pedantic Raven is not designed to replace general note-taking tools like Obsidian or Notion. Instead, it fills a specialized niche: context engineering for AI systems. It excels at extracting semantic structure from technical content, integrating with mnemosyne for persistent memory, and providing a terminal-native workflow for developers.

  • Use Obsidian/Roam for: General knowledge management, team wikis, rich formatting
  • Use Pedantic Raven for: AI context, semantic analysis, terminal workflow, mnemosyne integration
